# Fleet Manager(フリート管理アシスタント)

[English](./README.md) | **日本語**

複数車両の管理のためのAIスキルです。車両群をまたいだ整備スケジュール、コスト追跡とTCO分析、稼働停止(ダウンタイム)の計画、ドライバー点検ルーチン、入れ替え時期の判断を支援します。

## 説明

フリート管理アシスタントは、1台分の整備の考え方を2台から200台へスケールさせます:全車が同時に止まらないよう整備をずらす、km単価を追跡して問題車両をあぶり出す、不具合を早期に拾うドライバー点検ルーチンを作る、そして「直し続ける」より「入れ替える」が正解になる瞬間を見極める。配送バン、社用車、職人の作業車・トラックはもちろん、複数台を持つ家庭にもそのまま使えます。

**対象:** 中小企業のオーナーと車両管理担当者、作業車を運用する個人事業主、複数台を管理する家庭

## できること

- **スケジューリング** — 複数車両の整備カレンダー作成、大きな整備をずらして稼働率を守る、車両単位・入庫単位で作業をまとめて入庫回数を減らす、季節作業(タイヤ、車検・点検)のバッチ処理化
- **コスト管理** — km単価(円/km)の追跡の仕組み化、固定費と変動費の分離、車両同士のベンチマークによる外れ値の発見、既知の整備予定に基づく予算予測、DIY/外注/メンテナンスパックの使い分け
- **ダウンタイム計画** — 単一障害点の特定(「棚が積めるバンは1台だけ」)、予備車両の損益計算、機械的な深刻度ではなく業務影響度による修理の優先順位付け
- **ドライバールーチン** — 車種に応じた毎日/毎週の運行前点検チェックリスト、実際に使われる不具合報告の習慣づくり、整備に詳しくないドライバー向けの油脂・タイヤ点検ルーチン
- **記録** — 保証請求・リセール価値・コスト分析を守る最小限の記録システム(スプレッドシートレベル)の設計、整備記録1件に必要なデータ項目
- **ライフサイクル判断** — 老朽車両の修理継続 vs 入れ替えの分析、コストカーブに基づく最適な入れ替え時期、フリート用途での購入時の考慮点(部品の共通化、ディーラー網)、売却時期と準備
- **コンプライアンスの意識付け** — 車検・点検期限、積載と免許の基本、記録保存義務(一般論として指摘。詳細は地域のルールによる)
- **混成フリート** — ガソリン・ディーゼル・EVが混在する車両群:整備パターンの違い、給油/充電の物流、ドライバー教育の要点

## プロンプト例

```text
配送バン5台(ディーゼル3台、EV2台)を各年間4万km運用しています。
常に稼働停止が1台以下になる整備カレンダーを作ってください。
```

```text
ドライバーが実際に続けられる5分間の毎日の運行前点検チェックリストと、
当日中に私へ届く不具合報告フォーマットを設計してください。
```

```text
3号車の今年の修理費がフリート平均の4倍です。履歴を貼ります:[貼付]。
これはハズレ車のパターンですか、それとも不運ですか?直すか、換えるか?
```

```text
8台のコストを追跡してkm単価の比較と傾向把握ができるスプレッドシートの
構造を作りたい。必要な列は何ですか?
```

```text
年式も走行距離も違う3台を家族で使っています。1つの季節整備計画に
まとめて、共通の予算見通しを立ててください。
```

## 推奨ワークフロー

1. **フリートの棚卸しをする。** 各車両:年式、メーカー、車種、燃料種別、現在の走行距離、年間走行距離、役割、既知の状態。
2. **運用上の制約を伝える。** 同時に何台まで止められるか、予算のサイクル、誰が運転するか、自社整備の能力。
3. **カレンダーとチェックリストから始める。** この2つの成果物が最も速く価値を出します。
4. **毎月データを戻す。** 走行距離とコストを更新し、履歴が溜まったら外れ値と傾向の分析を依頼します。
5. **修理費が車両価値の約30〜50%を超えそうなときは**、作業を承認する前に修理 vs 入れ替えの分析を回してください。
6. **年に一度見直す。** フリート構成、入れ替えパイプライン、記録システムが実際に維持されているか。

## 制限事項

- **永続的なデータベースはありません。** アシスタントはセッションごとに貼り付けられた記録で動きます — 設計してもらった記録システムこそが本当の記憶装置です。
- **費用の数字は目安です。** 工賃・部品価格・リセール価値は地域と時期で変動します。可能な限り自社の実績値を使ってください。
- **法令要件は地域依存です**(事業用車両の点検制度、運行記録、免許区分)— スキルはカテゴリを指摘し、要件は行政が定めます。
- **大規模フリートの車両管理システムの代替ではありません。** スプレッドシート規模の運用を補完し、専用ソフトへ移行すべきタイミングの判断を助けます。
- **個別車両の故障診断は他のスキルの担当です** — 症状は [Vehicle Troubleshooting](../vehicle-troubleshooting/README.ja.md) や [OBD-II Diagnostic Assistant](../obd2-diagnostic-assistant/README.ja.md) に引き継いでください。
